# n8n-git-sync

> This workflow synchronizes n8n workflows with a GitHub repository, creating or updating workflow JSON files and optionally generating README.md documentation using AI.

---

## 📌 Overview

This n8n workflow automates the process of maintaining n8n workflow definitions in a GitHub repository. It retrieves all active workflows from the n8n instance, then compares them against their corresponding files in a designated GitHub repository. If a workflow is new or has been modified in n8n, the workflow will either create a new JSON file in GitHub or update an existing one.

Additionally, the workflow includes an optional feature to generate and manage `README.md` documentation for each synchronized workflow. This documentation is created or updated using an AI model, ensuring that the GitHub repository not only contains the workflow definitions but also clear, automatically generated explanations for each.

---

## 🎯 Purpose

The primary purpose of this workflow is to provide version control and collaborative capabilities for n8n workflows by integrating them with GitHub. It ensures that workflow definitions are backed up, trackable, and easily shareable. The optional AI-powered documentation generation further enhances maintainability and understanding for other developers, making it easier to onboard new team members or review existing automations.

---

## 📥 Inputs

*   **Manual Trigger**: Initiates the workflow execution.
*   **Configuration**: Defined within the workflow, specifying:
    *   GitHub repository details (owner, repository name, branch, workflow folder).
    *   Documentation settings (enabled/disabled, AI provider, approval method).
    *   AI model prompt and user prompt for README generation.

---

## 📤 Outputs

*   Updated or newly created `.json` files for n8n workflows in the specified GitHub repository.
*   Optionally, updated or newly created `README.md` files for each workflow in the GitHub repository.

---

## ⚙️ Requirements

*   **n8n API Credential**: Required to access and retrieve workflow definitions from the n8n instance. (e.g., `n8n account`)
*   **GitHub OAuth2 API Credential**: Required to interact with the GitHub repository (read, create, update files). (e.g., `GitHub account`)
*   **Google Gemini (PaLM) API Credential**: Required for AI-powered README generation if documentation is enabled. (e.g., `Google Gemini(PaLM) Api account 2`)
