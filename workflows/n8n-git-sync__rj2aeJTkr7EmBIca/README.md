# n8n-git-sync

> This workflow synchronizes n8n workflows with a GitHub repository, creating or updating workflow files and optionally generating READMEs.

---

## 📌 Overview

This workflow automates the synchronization of n8n workflows with a specified GitHub repository. It retrieves all active workflows from the n8n instance, prepares their data, and then checks for their existence in the configured GitHub repository.

If a workflow file is found in GitHub, the workflow compares its content with the current n8n workflow definition. If changes are detected, the GitHub file is updated, and the update is logged to a Google Sheet. If no changes are found, no action is taken.

If a workflow file does not exist in GitHub, the workflow creates a new file for it. Optionally, it can also generate a basic README.md file for the newly created workflow using an AI model.

---

## 🎯 Purpose

The primary purpose of this workflow is to maintain a version-controlled backup of n8n workflows in a GitHub repository. This ensures that workflow definitions are tracked, changes are logged, and a historical record is kept, facilitating collaboration, disaster recovery, and adherence to best practices for code management.

## 📥 Inputs

- **Manual Trigger**: The workflow is initiated manually.
- **Configuration**: Internal JSON configuration specifies GitHub repository details (owner, repository, branch, workflow folder) and documentation generation settings (enabled, provider, approval).

---

## 📤 Outputs

- **GitHub Workflow Files**: Updated or newly created workflow JSON files in the specified GitHub repository.
- **GitHub README.md Files**: Optional README.md files generated in GitHub for new workflows.
- **Google Sheet Log**: Log entries in a Google Sheet detailing workflow updates.

---

## ⚙️ Requirements

- **n8n API Credentials**: Required to retrieve workflow definitions from the n8n instance.
- **GitHub OAuth2 API Credentials**: Required to interact with the GitHub repository (get, create, edit files).
- **Google Sheets OAuth2 API Credentials**: Required to append rows to the specified Google Sheet for logging.
- **Google Gemini (PaLM) API Credentials**: Required if the documentation generation feature is enabled to create README.md files.
