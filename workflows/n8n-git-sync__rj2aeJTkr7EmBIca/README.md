# n8n-git-sync

> Synchronizes n8n workflows with a GitHub repository, providing version control and optional documentation generation.

---

## 📌 Overview

This workflow automates the synchronization of n8n workflows with a specified GitHub repository. It retrieves all active workflows from the n8n instance and compares them against their corresponding files in GitHub. If a workflow is new, it creates a new JSON file in the repository. If an existing workflow has been modified in n8n, the workflow updates the GitHub file to reflect the latest version.

Beyond basic synchronization, this workflow also includes functionality to optionally generate a README.md file for newly created workflows, either as a basic placeholder or using an AI model for more comprehensive documentation. It also logs workflow updates by appending a row to a Google Sheet.

This automation ensures that your n8n workflows are continuously backed up and version-controlled in GitHub, facilitating collaboration and disaster recovery.

---

## 🎯 Purpose

The primary purpose of this workflow is to provide robust version control and backup for n8n workflows by synchronizing them with a GitHub repository. This ensures that all workflow definitions are stored externally, enabling tracking of changes, easy rollback to previous versions, and collaboration among developers. Additionally, it automates the creation of basic documentation for new workflows, streamlining the development and maintenance process.

## 📤 Outputs

- **GitHub Repository Files**: Creates or updates `.json` files representing n8n workflows in the configured GitHub repository.
- **GitHub README.md Files**: Optionally creates or updates `README.md` files for workflows in the GitHub repository.
- **Google Sheet Rows**: Appends data to a Google Sheet, typically logging details of workflow updates.

---

## ⚙️ Requirements

- **n8n API Credentials**: Required to access and retrieve workflow definitions from the n8n instance.
- **GitHub OAuth2 Credentials**: Required to authenticate with GitHub for file operations (get, create, edit).
- **Google Sheets OAuth2 Credentials**: Required to append data to the specified Google Sheet.
- **Google Gemini (PaLM) API Credentials**: Required if the AI-powered README generation feature is enabled.
