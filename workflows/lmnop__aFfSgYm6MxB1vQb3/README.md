# GitSync for n8n

> Automatically synchronizes n8n workflows with a GitHub repository, creating or updating workflow JSON files and generating README documentation.

---

## 📌 Overview

This workflow monitors changes to n8n workflows within the n8n instance. When a workflow is updated, it retrieves the latest definition and compares it against its corresponding file in a specified GitHub repository.

If the workflow does not exist in GitHub, the workflow creates a new JSON file for it in the repository. If the workflow already exists but has changed, the workflow updates the existing file in GitHub.

Additionally, for newly created workflows, this workflow generates a basic README.md file and pushes it to the repository alongside the workflow JSON.

---

## 🎯 Purpose

The primary purpose of this workflow is to maintain a synchronized version of n8n workflows in a GitHub repository. This enables version control, collaboration, and automated documentation for n8n workflows, facilitating better development practices and disaster recovery.

## 📝 Notes

-   The workflow uses `Function` nodes to clean and normalize workflow JSON data before comparison and storage. This ensures that only relevant structural and functional changes trigger updates, ignoring dynamic fields like `id`, `createdAt`, `updatedAt`, `position`, etc.
-   The target GitHub repository and organization (`n8n-io/n8n-workflows`) are hardcoded in the HTTP Request nodes. These may need to be adjusted if you intend to use a different repository.
-   The "Generate README" node creates a basic README template. This template can be customized within the node's code to fit specific documentation standards.
